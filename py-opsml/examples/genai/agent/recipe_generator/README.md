# Music Recommender: Prompt Chaining Agent

This example shows how OpsML can be integrated into Google ADK for seamless integration into our metadata, monitoring and observability systems.
For general information on the Prompt Chaining Agent see [here](../prompt_chaining/Readme.MD)

## What OPSML Adds to Prompt Chaining

This example demonstrates OPSML's service and prompt registry capabilities, enabling production-grade versioning, reproducibility, and observability for prompt-based agents.

### Key OPSML Features

#### 1. CLI-Based Prompt and Service Registration

Define prompts as YAML files and register them with a single command:

```bash
# Lock: Creates service artifacts and registers prompts to OPSML
uv run opsml lock
```

This command:
- Reads `opsmlspec.yaml` to discover all prompts
- Registers each prompt as a `PromptCard` in the OPSML registry
- Creates a local service artifact in `opsml_service/` for reproducibility
- Generates `card_map.json` linking prompt aliases to registry versions

**Directory Structure:**
```
prompt_chaining_opsml/
├── opsmlspec.yaml           # Service definition
├── prompts/                 # Prompt definitions (YAML)
│   ├── pattern.yaml
│   ├── preference.yaml
│   ├── recommendation.yaml
│   └── search.yaml
└── opsml_service/           # Generated by 'lock' command
    ├── card_map.json        # Maps aliases → registry versions
    └── [prompt_name]/       # Local prompt artifacts
```

#### 2. Downloadable Agent Services

Registered agents can be downloaded and reused across environments:

```bash
# Install: Downloads a registered agent service (must be in same directory as opsmlspec.yaml)
uv run opsml install service
```

This enables:
- **Team Collaboration**: Share versioned agent services across team members
- **Environment Consistency**: Same prompts in dev, staging, and production
- **Reproducibility**: Recreate exact agent behavior from any point in time

#### 3. Runtime Prompt Loading via AppState

Load registered prompts directly in your agent code:

```python
from opsml.app import AppState
from pathlib import Path

# Load the service and all registered prompts
app = AppState.from_path(path=Path("opsml_service"))

# Access prompts by their alias (defined in opsmlspec.yaml)
pattern_prompt = app.service["pattern_prompt"]
preference_prompt = app.service["preference_prompt"]
recommendation_prompt = app.service["recommendation_prompt"]
search_prompt = app.service["search_prompt"]
```

**Benefits:**
- **Zero Hardcoding**: Prompts live in registry, not scattered in code
- **Hot Swapping**: Update prompts without code deployments
- **Version Control**: Roll back to previous prompt versions if needed

#### 4. Seamless Integration with ADK Agents

Populate Google ADK agents with OPSML-managed prompts:

```python
from agent.lifespan import prompts  # Loads from AppState

# Create agents using versioned prompts
preference_analyzer = Agent(
    model=LiteLlm(model=prompts.preference.prompt.model),
    name="PreferenceAnalyzer",
    instruction=prompts.preference.prompt.messages[0].text,  # From registry
    tools=[get_listening_history],
    output_key="user_preferences",
)

pattern_recognizer = Agent(
    model=LiteLlm(model=prompts.pattern.prompt.model),
    name="PatternRecognizer",
    instruction=prompts.pattern.prompt.messages[0].text,  # From registry
    output_key="search_strategy",
)
```

#### 5. UI-Based Observability (Current and Future)

- **Lineage Tracking**: Trace which prompts were used in which agent runs
- **A/B Testing**: Compare performance across prompt versions (evaluations coming soon!)

### The OPSML Advantage

| Without OPSML | With OPSML |
|---------------|------------|
| Prompts hardcoded in Python strings | Prompts versioned as YAML in registry |
| No prompt version tracking | Full lineage and rollback capability |
| Manual copy-paste to share agents | Single command to install agent services |
| No UI visibility into prompts | Browse, compare, and manage prompts in UI |
| Manual testing of prompt changes | Automated evaluation pipelines (future) |
