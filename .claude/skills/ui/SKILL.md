description: The UI skill contains components and utilities for building the OpsML user interface, including trace visualization, card browsing, and more.
---


When editing UI components, please keep the following in mind:
- The UI is built with SvelteKit and Tailwind CSS. Follow existing patterns for styling and component structure.
- Focus on maintainability and readability. The UI code should be easy to understand and modify by other developers.
- Ensure that any new components or changes are well-documented and tested. Include comments in the code where necessary to explain complex logic or design decisions.
- When making changes to the UI, consider the user experience and aim for a clean, intuitive interface that aligns with the overall design of OpsML.
- Refer to [rules.md](rules.md) for guidelines on component design and best practices.
